# 功能规格

## 结论

第一版规格是一个后台输入法辅助工具。它只负责在“外部语音工具未运行”的情况下，用右 Ctrl 协调豆包语音输入和鼠须管恢复。

## 术语

- 主输入法：鼠须管（Rime / Squirrel）。
- 豆包输入法：`/Library/Input Methods/DoubaoIme.app` 对应的输入法。
- 外部语音工具：默认是 `/Applications/Type4Me.app`，路径应可配置。
- 右 Ctrl：键盘右侧 Control 修饰键。
- 豆包语音快捷键：豆包输入法设置中“语音输入 -> 免按模式”的快捷键。当前用户设定为“右 Ctrl”。
- 接管右 Ctrl：本工具把右 Ctrl 作为豆包语音开关处理。第一版采用旁听策略，不吞掉物理右 Ctrl。
- 暂停接管：本工具不处理右 Ctrl，让其他工具使用。

## 配置规格

第一版至少包含以下配置项：

- `externalVoiceAppPath`：外部语音工具路径，默认值为 `/Applications/Type4Me.app`。
- `externalVoiceBundleId`：外部语音工具 bundle id，默认值为 `com.type4me.app`。
- `primaryInputSourceId`：主输入法输入源，默认值为鼠须管简体 `im.rime.inputmethod.Squirrel.Hans`。
- `doubaoInputSourceId`：豆包输入源，默认值为 `com.bytedance.inputmethod.doubaoime.pinyin`。
- `doubaoVoiceHotkey`：豆包语音快捷键，第一版只支持 `rightControl`，必须与豆包“免按模式”的设置一致。

配置项用于判断外部语音工具是否正在运行、切换输入源，并匹配豆包语音快捷键。

第一版可以暂时不提供图形配置界面，但配置位置和修改方式必须在设计阶段明确。

## 状态规格

工具运行时至少区分以下状态：

- `idle`：普通状态，本工具可以接管右 Ctrl。
- `suspended`：外部语音工具正在运行，本工具暂停接管右 Ctrl。
- `doubaoVoiceActive`：本工具已尝试进入豆包语音输入流程。
- `error`：出现无法继续的错误，例如缺少权限或找不到输入源。

状态切换规则：

- 当外部语音工具正在运行时，工具应进入 `suspended`。
- 当外部语音工具退出后，工具应离开 `suspended`，恢复到可接管右 Ctrl 的状态。
- 当用户在 `idle` 下按右 Ctrl，工具进入 `doubaoVoiceActive`。
- 当用户在 `doubaoVoiceActive` 下再次按右 Ctrl，工具应恢复主输入法，并回到 `idle`。

## 外部语音工具让渡规格

当 `externalVoiceAppPath` 对应的应用正在运行时：

1. 用户按右 Ctrl。
2. 本工具不得切换到豆包输入法。
3. 本工具不得恢复鼠须管输入法。
4. 本工具不得模拟任何用于豆包语音的快捷键。
5. 本工具应保持暂停接管状态，直到外部语音工具退出。

验收标准：

- Type4Me 正在运行时，右 Ctrl 的行为不被本工具改变。
- Type4Me 退出后，右 Ctrl 可重新用于豆包语音流程。

## 进入豆包语音规格

当外部语音工具未运行，且工具处于 `idle` 时：

1. 用户按右 Ctrl。
2. 工具识别这是 `doubaoVoiceHotkey`，而不是左 Ctrl。
3. 工具延迟处理这次事件，让物理右 Ctrl 先被当前鼠须管状态消化。
4. 工具切换当前输入法到豆包输入法。
5. 工具等待系统实际报告当前输入源已经是豆包。
6. 工具按豆包设置中的同一快捷键触发豆包语音输入。
7. 工具进入 `doubaoVoiceActive`，保持豆包语音流程继续运行。

验收标准：

- 左 Ctrl 不触发豆包语音流程。
- 右 Ctrl 触发豆包语音流程。
- 豆包只应收到一次有效的右 Ctrl 触发，不应出现重复触发导致的“语音唤起方式调整”。
- 触发后，用户可以开始使用豆包语音输入。

## 退出提交规格

当用户第二次按右 Ctrl 退出豆包语音流程：

1. 工具等待豆包把识别结果提交到输入框。
2. 工具恢复鼠须管。
3. 工具不得再次向豆包发送右 Ctrl 停止快捷键。

验收标准：

- 正常短按流程不受影响。
- 第二次右 Ctrl 后，已识别文字应优先上屏，不应被切换输入法或 `Esc` 打断。

## 退出豆包语音规格

当工具处于 `doubaoVoiceActive` 时：

1. 用户再次按右 Ctrl。
2. 工具尽快切换当前输入法到鼠须管。
3. 工具回到 `idle`。

验收标准：

- 第二次右 Ctrl 后，当前输入法必须恢复为鼠须管。
- 第二次右 Ctrl 不再额外向豆包发送停止快捷键，也不自动发送 `Esc`，避免取消豆包尚未提交的识别结果。

## 空闲守护规格

当工具处于 `idle`，且外部语音工具未运行时：

1. 工具应定期检查当前输入法。
2. 如果当前输入法不是鼠须管，工具应自动切回鼠须管。
3. 如果 Type4Me 等外部语音工具正在运行，工具不得强行恢复鼠须管。

验收标准：

- 语音流程结束后即使一次恢复失败，工具也应在空闲轮询中把输入法拉回鼠须管。
- 工具不应干扰外部语音工具运行时的右 Ctrl 行为。

## 输入源识别规格

工具不得只根据 `.app` 路径硬编码输入源。

工具需要在运行或初始化阶段确认：

- 鼠须管输入源存在。
- 豆包输入源存在。
- 鼠须管输入源可被系统选择。
- 豆包输入源可被系统选择。
- 能够把输入法切换到鼠须管。
- 能够把输入法切换到豆包。

验收标准：

- 找不到鼠须管时，工具进入错误状态并提示原因。
- 找不到豆包时，工具进入错误状态并提示原因。
- 输入源已安装但未启用时，工具进入错误状态并提示需要用户手动启用。
- 切换失败时，工具进入错误状态或输出明确错误信息。

## 权限规格

如果工具需要 macOS 权限，应只请求必要权限。

可能需要的权限包括：

- 辅助功能权限。
- 输入监听权限。
- 自动化权限。

验收标准：

- 权限不足时，工具能明确提示缺少什么权限。
- 权限不足时，工具不得静默伪装成正常运行。
- 工具不得修改系统设置来绕过权限。

## 启动和退出规格

启动时：

1. 加载配置。
2. 检查外部语音工具运行状态。
3. 检查必要输入源。
4. 检查必要权限。
5. 开始监听右 Ctrl。

退出时：

1. 停止监听右 Ctrl。
2. 如果工具认为自己处于 `doubaoVoiceActive`，应尽力恢复鼠须管。
3. 不修改输入法列表，不修改系统输入法配置。

## 菜单栏控制规格

正式试用版应提供菜单栏图标，避免用户必须记忆终端命令。

菜单栏至少包含：

- 当前状态：运行中、已暂停、Type4Me 运行中已让渡、豆包语音中或错误。
- `暂停` / `启用`：暂停或恢复右 Ctrl 接管。
- `恢复鼠须管`：立即尝试把当前输入法切回鼠须管。
- `退出`：关闭后台进程。

验收标准：

- 用户可以通过菜单栏退出程序。
- 用户可以通过菜单栏暂停程序，暂停后不再监听右 Ctrl。
- 用户可以通过菜单栏重新启用程序。
- 用户可以通过菜单栏恢复鼠须管。

## 日志和反馈规格

第一版至少需要开发者可见的反馈，例如标准输出日志。

日志至少覆盖：

- 配置加载结果。
- 外部语音工具运行状态变化。
- 输入源识别结果。
- 权限检查结果。
- 右 Ctrl 触发进入豆包语音。
- 右 Ctrl 触发恢复鼠须管。
- 切换失败或触发失败。

## 非功能规格

- 工具应能在后台运行。
- 工具不应明显影响普通键盘输入。
- 工具不应依赖网络。
- 工具不应修改豆包或鼠须管应用内部文件。
- 工具不应安装全局依赖。
- 工具不应默认写入系统级配置目录。

## 暂不纳入第一版

- 开机自启动。
- 图形配置界面。
- 多个外部语音工具同时配置。
- 完整输入法管理。
- 自动安装或配置 macOS 权限。
